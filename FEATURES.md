## ١. لوحة تحكم التاجر (Merchant Dashboard)

### ١.١ نظرة عامة
- إجمالي الطلبات، الإيرادات، المنتجات، الطلبات المعلقة
- رسم بياني للمبيعات (أسبوعي/شهري) — PRO+
- أفضل 5 منتجات مبيعاً — PRO+
- آخر 5 طلبات

### ١.٢ إدارة المنتجات
- **إضافة/تعديل/حذف** منتجات مع صور متعددة
- **الحقول:** اسم، وصف، سعر، سعر مقارنة (شطب)، مخزون، فئة
- **وحدات:** قطعة، كيلو، لتر، متر... مع ملصق مخصص بالعربية
- **SEO:** عنوان، وصف، slug مخصص لكل منتج
- **الحالة:** نشط / مخفي
- **البحث والفلترة** حسب الفئة والحالة والسعر

### ١.٣ إدارة الفئات
- إنشاء وتعديل وحذف الفئات
- ربط المنتجات بفئات
- slug تلقائي من اسم الفئة

### ١.٤ إدارة الطلبات
- عرض جميع الطلبات مع التفاصيل الكاملة
- تحديث الحالة: معلق ← مدفوع ← شُحن ← تم التسليم / ملغي
- بيانات الشحن: الاسم، العنوان، المدينة، الهاتف
- تصفية حسب الحالة والتاريخ
- **تصدير CSV** — PRO+

### ١.٥ إعدادات المتجر
- اسم المتجر، الوصف، الشعار
- اللون الرئيسي للمتجر (color picker)
- العملة: IQD، USD، SAR، AED
- نوع المتجر: عام، أزياء، إلكترونيات، مأكولات، مستحضرات
- رابط المتجر العام: `/store/[slug]`

### ١.٦ تصميم المتجر (Builder)
- **3 قوالب:** بسيط (Minimal)، جريء (Bold)، أنيق (Elegant)
- بانرات ترويجية مخصصة (عنوان، صورة، لون خلفية، رابط)
- ترتيب البانرات

### ١.٧ ترقية الخطة
- عرض الخطط المتاحة مع المقارنة
- إرسال طلب ترقية مع إيصال الدفع
- تتبع حالة الطلبات السابقة

### ١.٨ كوبونات الخصم — PRO+
- إنشاء كوبونات بنسبة مئوية أو قيمة ثابتة
- تحديد الحد الأدنى للطلب
- تحديد عدد مرات الاستخدام وتاريخ انتهاء الصلاحية
- تفعيل/تعطيل الكوبون

### ١.٩ إدارة الموظفين — PRO+
- إضافة موظفين بصلاحيات محددة
- أدوار: مدير مبيعات، موظف مستودع، دعم عملاء
- تفعيل/تعطيل حسابات الموظفين

### ١.١٠ برنامج الإحالة — PRO+
- رابط إحالة مخصص لكل تاجر
- تتبع التحويلات والعمولات
- عرض الأرباح المتراكمة

### ١.١١ الذكاء الاصطناعي
- **كتابة وصف منتج** بالذكاء الاصطناعي
- **اقتراح عنوان SEO** للمنتج
- **تحليل المنافسين** واقتراح أسعار
- عداد الطلبات المتبقية حسب الخطة

### ١.١٢ Widget للمواقع الخارجية — PRO+
- كود تضمين جاهز (`<script>`)
- إدارة النطاقات المسموح بها
- اختيار الثيم: فاتح / داكن
- حد معدل الطلبات قابل للضبط

## ٢. واجهة المتجر العام (Storefront)

### ٢.١ الصفحة الرئيسية
- بانرات ترويجية متحركة
- شبكة المنتجات (2–4 أعمدة حسب الشاشة)
- فلترة حسب الفئة
- البحث في المنتجات
- زر "إضافة إلى السلة" مباشر من الشبكة

### ٢.٢ صفحة المنتج
- معرض صور (أول صورة، مصغرات)
- السعر والسعر قبل الخصم (مشطوب)
- وصف مفصل
- اختيار الكمية مع حد المخزون
- زر الإضافة للسلة مع تأكيد بصري
- حالة "نفد من المخزون"

### ٢.٣ سلة التسوق
- إضافة/تعديل/حذف المنتجات
- حساب الإجمالي لحظياً
- دعم الكوبون (مع عرض قيمة الخصم) — PRO+
- زر "إتمام الطلب"

### ٢.٤ صفحة الدفع والشراء
- نموذج بيانات التوصيل (الاسم، العنوان، المدينة، الهاتف)
- طرق الدفع:
  - **تحويل بنكي** (QR code + تفاصيل الحساب)
  - **Stripe** (بطاقة ائتمانية) — اختياري
- ملخص الطلب مع صور المنتجات

### ٢.٥ صفحة تأكيد الطلب
- رقم الطلب، الإجمالي، ملخص المنتجات
- رسالة شكر
- زر "متابعة التسوق"

### ٢.٦ دردشة العملاء
- نافذة دردشة مدمجة في المتجر
- ردود تلقائية بالذكاء الاصطناعي (بناءً على منتجات المتجر)
- إشعار للتاجر بالرسائل الجديدة

### ٢.٧ متجر معطل
- صفحة "المتجر غير متاح حالياً" عند التعليق
- عرض سبب التعليق (إذا وُجد)

## ٥. لوحة تحكم السوبر أدمن

### ٥.١ لوحة المؤشرات
- إجمالي التجار، المتاجر، الطلبات، الإيرادات
- تجار جدد هذا الشهر
- رسوم بيانية تفاعلية

### ٥.٢ إدارة التجار
- قائمة جميع التجار مع البحث والفلترة
- تفعيل/تعطيل الحسابات
- تغيير الخطة (FREE ← PRO ← ENTERPRISE)
- حذف الحساب مع جميع بياناته
- عرض رقم الواتساب والتواصل المباشر

### ٥.٣ إدارة المتاجر
- عرض جميع المتاجر
- تفعيل/تعطيل متجر (يظهر صفحة تعليق للعملاء)
- دخول وضع المحرر (Editor Mode) — تعديل المتجر بدلاً من التاجر

### ٥.٤ إدارة الطلبات
- عرض جميع طلبات المنصة
- فلترة حسب المتجر، الحالة، التاريخ
- تصدير CSV

### ٥.٥ طلبات الاشتراك (Payments)
- عرض طلبات ترقية الخطة
- قبول أو رفض مع ملاحظة للتاجر
- عرض إيصال الدفع
- التواصل عبر واتساب
- إشعار فوري للتاجر عند الموافقة/الرفض

### ٥.٦ التحليلات
- إيرادات المنصة يومياً (آخر 30 يوم)
- أفضل 10 متاجر حسب عدد الطلبات
- رسم بياني للمبيعات

### ٥.٧ سجل الأحداث (Audit Log)
- جميع إجراءات الأدمن (من، ماذا، متى، IP)
- أنواع السجلات: تعديل خطة، تعطيل حساب، حذف، مراجعة دفع...

### ٥.٨ إعدادات الذكاء الاصطناعي
- عرض استخدام كل تاجر للـ AI
- تعديل الحد الشهري لكل تاجر

### ٥.٩ إدارة المارketplace
- الموافقة على إدراج المتاجر في السوق
- تمييز منتجات (Featured)
- إدارة معدلات العمولة

## ٦. الإشعارات

### للتاجر
- ✅ **موافقة على الخطة** — مع قائمة كاملة بالمزايا الجديدة + دليل البدء
- ❌ **رفض طلب الترقية** — مع ملاحظة الأدمن
- 🛒 **طلب جديد** — عند وصول طلب للمتجر
- 💬 **رسالة عميل** — في دردشة المتجر

### للأدمن
- 💳 **طلب اشتراك جديد** — عند إرسال تاجر طلب ترقية

### آلية التسليم
- polling تلقائي كل 30 ثانية
- شارة حمراء بعدد غير المقروءة
- قراءة فردية أو جماعية ("قراءة الكل")

## ٧. الأمان والتقنية

### ٧.١ المصادقة
- JWT tokens بمدة 7 أيام (قابل للتخصيص)
- تشفير كلمات المرور بـ bcrypt (12 rounds)
- "ابقني مسجلاً" — 30 يوماً في localStorage
- Google OAuth 2.0

### ٧.٢ متطلبات كلمة المرور
- 8 أحرف كحد أدنى
- يجب أن تحتوي على حرف كبير
- يجب أن تحتوي على حرف صغير
- يجب أن تحتوي على رقم

### ٧.٣ Rate Limiting
| المسار | الحد | النافذة الزمنية |
|--------|------|----------------|
| `/api/auth/*` | 5 محاولات (فاشلة فقط) | 15 دقيقة |
| `/api/storefront/*` | 120 طلب | دقيقة واحدة |
| `/api/*` (عام) | 200 طلب | دقيقة واحدة |

### ٧.٤ ترويسات الأمان
- Helmet.js (XSS، HSTS في الإنتاج، noSniff...)
- CORS محدود بنطاقات محددة
- X-Content-Type-Options: nosniff على الصور

### ٧.٥ رفع الملفات
- أنواع مسموح بها: JPEG, PNG, WebP, GIF
- الحد الأقصى: 5 MB
- التحقق من Magic Bytes (ليس MIME فقط)
- أسماء ملفات عشوائية (UUID)
- حظر directory listing

## ٨. دليل الرفع على السيرفر

### المتطلبات
- Node.js 18+
- MySQL 8.0+ أو MariaDB 10.6+
- Nginx (كـ reverse proxy)
- PM2 (لإدارة العمليات)

### الخطوات
```bash
# 1. نسخ الكود
git clone <repo>
cd bazar

# 2. إعداد المتغيرات
cp .env.example .env
# عدّل .env بالقيم الحقيقية

# 3. توليد JWT_SECRET قوي
openssl rand -base64 48

# 4. تثبيت التبعيات
npm install

# 5. بناء قاعدة البيانات
npm run db:generate
npm run db:migrate
npm run db:seed   # ينشئ حساب السوبر أدمن

# 6. بناء الواجهة
npm run build

# 7. تشغيل الخوادم
pm2 start ecosystem.config.js

# 8. Nginx config
# راجع nginx.conf المرفق
```

### إعداد Nginx المقترح
```nginx
server {
    listen 443 ssl http2;
    server_name api.yourdomain.com;

    ssl_certificate     /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;

    # Security headers
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload" always;
    add_header X-Frame-Options DENY always;
    add_header X-Content-Type-Options nosniff always;

    location / {
        proxy_pass http://127.0.0.1:4000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Upload size limit
    client_max_body_size 10M;
}
```

## ٩. الواجهات البرمجية (API Endpoints)

### Auth
| الطريقة | المسار | الوصف | المصادقة |
|---------|--------|--------|----------|
| POST | `/api/auth/register` | إنشاء حساب تاجر | — |
| POST | `/api/auth/login` | تسجيل الدخول | — |
| GET | `/api/auth/me` | بيانات المستخدم الحالي | JWT |
| POST | `/api/auth/google` | تسجيل دخول بـ Google | — |

### المتجر
| الطريقة | المسار | الوصف | المصادقة |
|---------|--------|--------|----------|
| GET | `/api/stores/my` | متجر التاجر | JWT Merchant |
| POST | `/api/stores` | إنشاء متجر | JWT Merchant |
| PUT | `/api/stores/:id` | تعديل المتجر | JWT Merchant |

### المنتجات
| الطريقة | المسار | الوصف | المصادقة |
|---------|--------|--------|----------|
| GET | `/api/products` | منتجات المتجر | JWT Merchant |
| POST | `/api/products` | إضافة منتج | JWT Merchant |
| PUT | `/api/products/:id` | تعديل منتج | JWT Merchant |
| DELETE | `/api/products/:id` | حذف منتج | JWT Merchant |

### الواجهة العامة (Storefront)
| الطريقة | المسار | الوصف | المصادقة |
|---------|--------|--------|----------|
| GET | `/api/storefront/:slug` | بيانات المتجر العام | — |
| GET | `/api/storefront/:slug/products` | منتجات المتجر | — |

> للقائمة الكاملة بجميع الـ endpoints راجع مجلد `apps/api/src/routes/`
